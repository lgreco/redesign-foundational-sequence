# Checklist

This is a list of items and priorities that Karima and Leo agree that need to be implemented in the proposed sequence of COMP 158 and COMP 159. (The sequence is refered as "158+159" sometimes).

Items and priorities are listed in random order.

* Explain and demonstrate the difference between *combinations* and *permutations.* Use both theoretical examples (e.g., powersef of a set) and practical/silly problems: how many $n$-scoops of icecream combinations can you make with Baskin-Robbin's 31 flavors? Permutations thereof?

* Discuss mathematical sets, their ability to "reject" (ok, ignore duplicates), and how this can be connected with primary keys, key-value pairs, hashtables, and Python dictionaries.

* COMP 158 should start within a *shell* (assume `bash`) reinforcing students skills in filesystem commands and plain text editor (our choice is `vim`). Eventually we'll transition to an IDE like VSC, but we need to build "muscle memory" and good habits at the CLI first.

* At some point the 158+159 sequence should cover basic probability and statistics. For example, compute $pi$ by throwing random points within a square with an inscribed circle.

* The current [outline for COMP 170](./sources/COMP%20170%20Spring%202025%20Course%20ScheduleV8-2.pdf) covers algorithms in week 2. We believe that's too early. We don't want to cover them so early in COMP 158. It's import to covert variables and expressions at depth first.

* We need to differentiate terminology between program i/o (`input()`, `open()`, etc) and function i/o. Maybe for functions/method use "entry" and "exit"?

The 158+159 sequence should be touching on the following items:

* terminal/shell 
* text /editor 
* file commanes 
* comp. organiz. basics (pyramic/memory hierarchy)
* IDE basics 
* Simple debugging 
* Variable types 
* Exprenions 
* Methods
* Lists, loops, 
* If statements
* Strings 
* Boolean
* Simple algoriths showcased as pseudocode
* Vector/matrices
* Exceptions
* Python file i/o
* Intro to object-oriented programming concepts
