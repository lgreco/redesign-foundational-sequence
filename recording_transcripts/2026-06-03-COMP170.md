## Strings, Numbers, and Number Systems (June 2)

With Bella's question, we uncovered a key distinction: the `*` operator on strings repeats text rather than multiplying. So `"22" * 3` produces `"222222"`, not `66`. To do arithmetic with a number stored as text, you first convert it with `int()`.

From there, we traced how positional number systems work. In decimal (base 10), each position is a power of 10: 2026 = 2 × 1000 + 0 × 100 + 2 × 10 + 6 × 1. Binary (base 2) follows the same rule using only 0 and 1. We introduced hexadecimal (base 16), which compresses four binary digits into one symbol (0–9, then A–F), and explained why uppercase letters have lower ASCII values than lowercase ones — early computers only had memory for uppercase; lowercase was added later.

## Loops and Scope (June 2–3)

We built toward loops by studying ASCII art triangles: printing a 6-row triangle by hand takes 6 separate print statements — typing, not programming. The key pattern: row *n* prints exactly *n* stars. That observation is the whole idea behind a loop.

In Python, loops use `for [variable] in range(...)`. The `range` function starts at 0 by default, goes up to but not including the stop value, and advances by 1 each step — so `range(5)` gives 0, 1, 2, 3, 4. Together with Kandace and Erica, we built the intuition for zero-based counting: the computer labels you "0" at the front of a line because zero people are ahead of you.

We explored *scope*: every indented line under a `for` statement runs once per loop iteration, like sub-items grouped under a shopping-list category. Moving a line back to the left margin puts it after the loop. Next week: `if` statements.
