## Logic and Decision-Making

We opened with a shopping-list story: "Go buy bread. If they have eggs, get a dozen." A literal reading produces twelve loaves of bread — the joke illustrates how the *scope* of a conditional matters. This week's theme is logic, Boolean expressions, and if statements — one of the four fundamentals of programming (alongside loops, lists, and strings).

## Decision Points and Flowcharts

We visualized branching using a diamond-shaped flowchart node with a yes/no question inside and two paths leading out. The umbrella example — "if it's raining, take an umbrella; either way, walk the dogs" — showed how both branches eventually rejoin. We wrote a short Python program asking the user whether it's raining, handling both uppercase `Y` and lowercase `y` as valid "yes" answers.

## Boolean Values

We introduced *Boolean* as a data type with exactly two possible values: `True` or `False`. Any question with a definitive yes-or-no answer is a Boolean expression — "Is it raining?", "Is Illinois next to Ohio?" Questions like "Is Chicago the finest city?" are not Boolean because the answer depends on opinion.

## AND and OR

We built truth tables for the two fundamental logic operators. OR is `True` whenever at least one part is true; AND requires both parts to be true. In Python, these are written in plain lowercase: `or` and `and`.

## Assignment vs. Equality

We distinguished `=` (assignment: stores a value into a variable) from `==` (comparison: tests whether two values are equal). String comparison is case-sensitive and space-sensitive — `"Leo"` and `"leo"` are not equal, and `"Leo"` and `"Leo "` (with a trailing space) are also not equal.

## Modulo and Integer Division

We introduced the modulo operator (`%`), which gives the remainder of integer division. Dividing any number by 2 leaves a remainder of 0 (even) or 1 (odd). A real-world application: alternating gray/white row shading in data tables uses `line_number % 2 == 0` to decide which color to apply. We previewed tomorrow's problem: using integer division and modulo to assign a linear list of passengers to two-dimensional airplane seats — a first glimpse of how programs translate flat data into structured layouts.
