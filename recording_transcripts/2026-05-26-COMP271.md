## Review: The Mississippi Problem

Revisited the block-letter printing problem from last week. Vertical printing (stacking letters on top of each other) is straightforward: write one method per letter containing a series of `print` statements. Printing horizontally is harder and motivates a more sophisticated approach.

## Horizontal Block Printing with Arrays

Instead of hardcoded print statements, each letter is stored as a **list of strings**, where each string represents one row of the block letter. To print a name horizontally, use nested loops:

- **Outer loop**: iterate over row index (0 to height)
- **Inner loop**: for each letter in the name, print `letter[row]`
- Print a blank line after each row

This avoids rewriting code for different names and generalizes to any word (e.g., Mississippi). The solution is on GitHub under `mississippi_horizontal`.

## What Is an Array?

An array is a **contiguous block of memory** holding elements of the **same data type**, accessed **randomly via an index**. Key properties:

- First element at base address; element `i` is at `base + i × sizeof(type)`
- Index out of bounds = crash (like trying to reach a floor that doesn't exist)
- Fixed size in static languages (Java, C, C++); cannot grow after creation

## Python Lists vs. True Arrays

Python does not have true arrays — it has **lists**, which are dynamic (elements can be added with `.append()`). For this course, we treat Python lists as arrays, but recognize the difference: real arrays cannot be resized after allocation.

## Why Arrays Matter

Arrays underlie nearly every computational problem, including large language models (LLMs). Understanding arrays is a prerequisite for almost any technical role in computing.
