## Arrays vs. Python Lists

We revisited the distinction between true arrays — fixed-size blocks of memory where every element is the same type — and Python lists, which can grow dynamically via `append`. In languages like Java, resizing an array requires significant effort; Python abstracts that complexity away, but understanding what happens underneath is exactly what this course is about.

## Intro to Object-Oriented Programming

We introduced the idea that a **class** is a blueprint for an **object** — a self-contained program with its own data and behavior. An object is created through **instantiation**, guided by a special method called the constructor (`__init__` in Python). `self` is the keyword that refers to the object itself within its own methods.

## Building a DynamicArray Class

We wrote a `DynamicArray` class that simulates a fixed-capacity array using a Python list:

- `__init__`: initializes a list of four `-1` values (sentinel for "empty") and a `size` counter starting at 0.
- `add_zip_code(value)`: checks `size < 4` before inserting; places the new value at index `self.size` and increments `size`.
- `__str__`: builds a string representation so `print()` shows the contents meaningfully.

We tested it interactively, filling all four slots and confirming a fifth addition is rejected.
