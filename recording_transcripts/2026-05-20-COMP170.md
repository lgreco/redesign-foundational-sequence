## Live Coding: Solving a Quadratic Equation

A method `solve_quadratic_equation(a, b, c)` was written in Vim to illustrate program structure:

- **Doc string** — triple-quoted block at the top; becomes the user manual extracted by documentation tools.
- **Inline comments** — hash-prefixed lines that explain the logic step by step; even "painfully obvious" comments are better than none.
- **Discriminant check** — an `if/else` branch decides whether real solutions exist (`b² - 4ac ≥ 0`). Branching logic is how programs make decisions, mirroring everyday rules ("if red light, don't cross").
- **`import`** — the `math` library's `sqrt` function was imported to handle square roots.

The program was noted as intentionally imperfect: it uses `print` inside the method rather than returning values. Good design keeps computation and display separate — internal methods should return data; only a final display method should print.

## Problem Decomposition

Complex problems are solved by breaking them into small, manageable methods — just as earning a degree is broken into 40-50 courses. This habit of decomposition is the essence of programming, and building it is the goal of the course starting next week.
